1. Static

This <div> has a default setting of:

  display: static;

It takes its place in the normal flow of the page.

Its z-index is set to 99, but this is ignored because its position is static.

2. Relative

This <div> has a custom setting of:

  display: relative;
  top: -3em;
  left: 75% /* of the width of its parent <main> */

It reserves its place in the normal flow: no other elements will move in to take its place.

The values for its top and left properties position it relative to its default position in the flow.

Its value for z-index is auto by default, but it appears above the red <div> because the z-index of the red <div> is ignored.

3. Static

This <div> has a default setting of:

  display: static;

It takes its place in the normal flow of the page.

4. Absolute

This <div> has a custom setting of:

  display: absolute;
  top: 83.333vh;
  left: 75%;
  z-index: 10;

It is removed from the normal flow of the page, so the cyan <div> moves up to take its place.

The values for its top and left properties position it with respect to its nearest parent for which the value of position is not static. In this case, this is the <main> element, which has a grey background.

It appears in front of the cyan <div> because it has a higher value for z-index.

5. Relative

This <div> has a custom setting of:

  display: relative;
  z-index: 2;

It takes its place in the normal flow of the page.

It appears behind the green <div>, because it has a lower value for z-index.

6. Sticky

(Scroll down)

This <div> has a default setting of:

  display: sticky;
  top: 20vh;
  left: 0px;
  width: 75%;
  height: 33vh;

It takes its place in the normal flow of the page until it reaches the position within the viewport indicated by the top or left attributes.

When this happens it behaves as if it were fixed, but it remains constrained to the area of its parent (<main>), which means that it can be dragged out of the viewport as its parent is scrolled out of sight.

It does not have a custom z-index property, so it will be hidden behind any <div> that appears further down the HTMLÂ page.

7. Fixed

This <div> is fixed to the bottom of the viewport with the custom settings: 

  position: fixed;
  bottom: 0;
  left: 0;
  width: 100vw;
  z-index: 5;

The display: fixed setting removes this <div> from the normal flow of the page, so the violet <div> moves up to fill the gap after the blue <div>.

The width setting ensures that it fills the full width of the viewport, rather than 100% of its parent`s width.

Its z-index setting places it in front of the cyan <div> but behind the green <div>.
